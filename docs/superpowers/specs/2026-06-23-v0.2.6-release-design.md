# Mochi v0.2.6 Stable Release

## Scope

Release the current `main` as `v0.2.6` with two user-visible corrections:

- Usage refreshes preserve the provider selected for the tray icon.
- CLI diagnostics reports the same version as the installed application.

## Release changes

- Set `0.2.6` in `package.json`, `src-tauri/Cargo.toml`, `src-tauri/Cargo.lock`, and `src-tauri/tauri.conf.json`.
- Update both copies of the release notes in the stable and unstable workflows with the same user-facing highlights.
- Make no unrelated product or tooling changes.

## Validation and delivery

Run every required local validation command, open a `chore/release-v0.2.6` pull request, and wait for every GitHub check to pass. Squash-merge the PR, delete its branch, tag the resulting `main` commit as `v0.2.6`, and push the tag. Monitor the stable release workflow until all jobs pass, then verify the published GitHub release and updater feed.

If any required check or release job fails, do not publish or claim completion; diagnose and fix it through a focused branch and PR.
