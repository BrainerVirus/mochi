# Mochi v0.2.6 Stable Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish Mochi `v0.2.6` from a fully validated `main` branch.

**Architecture:** Keep the release change metadata-only: synchronize all version sources and duplicate the same user-facing highlights in both release workflows. Deliver through GitHub Flow, then create the stable tag only after the release PR and every required check pass.

**Tech Stack:** Git, GitHub Actions, pnpm/Vitest/Vite/Oxc, Rust/Cargo, Tauri v2

---

Use the same highlights in `releaseBody` and the `release-notes` job arrays:

```markdown
Stable release that keeps your selected tray provider through usage refreshes and reports the installed version consistently in CLI diagnostics.

### What's changed

- **Tray selection stays put** — refreshing usage no longer replaces the selected provider icon with Overview.
- **Consistent version diagnostics** — the CLI now reports the same version as the installed Mochi app.
```

Use `Latest main build` instead of `Stable release` in the unstable workflow introduction.

### Task 3: Validate and merge the release PR

**Files:**

- Verify all files modified in Task 2.

- [ ] **Step 1: Run the complete local gate**

```bash
pnpm lint
pnpm format:check
CI=1 pnpm test
pnpm build
cargo fmt --manifest-path src-tauri/Cargo.toml -- --check
cargo clippy --manifest-path src-tauri/Cargo.toml --all-targets -- -D warnings
cargo test --manifest-path src-tauri/Cargo.toml --all-targets
```

Expected: every command exits zero.

- [ ] **Step 2: Commit and push**

```bash
git add package.json src-tauri/Cargo.toml src-tauri/Cargo.lock src-tauri/tauri.conf.json .github/workflows/release-stable.yml .github/workflows/release-unstable.yml
git commit -m "chore(release): bump version to 0.2.6"
git push -u origin chore/release-v0.2.6
```

- [ ] **Step 3: Open the release PR**

Open a ready PR titled `chore(release): bump version to 0.2.6` using `.github/PULL_REQUEST_TEMPLATE.md` and include every local validation command.

- [ ] **Step 4: Require green GitHub checks**

```bash
gh pr checks --watch --interval 10
```

Expected: every reported check passes. Do not merge on pending, skipped-required, cancelled, or failed checks.

- [ ] **Step 5: Squash-merge and delete the branch**

Squash-merge the PR, delete its remote branch, switch to `main`, fast-forward, and remove the local release branch.
