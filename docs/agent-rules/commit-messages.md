# Commit Message Conventions

When generating git commit messages, follow these rules:

- Use conventional commit format: `type(scope): description`.
- Use imperative mood: `Add feature`, not `Added feature`.
- Keep the subject line under 50 characters.
- Use valid types only: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `chore`, `ci`.
- Include scope when relevant, for example `api`, `ui`, `auth`, `tauri`, `ci`, or `docs`.
- If extra context is needed, add a body section with concise bullet points using `*`.
- Do not include generated-by footers or agent signatures.
